All packages that arrive at the Debian user's repository are compiled in a single codespace with a desktop environment, functioning as a dedicated lab for each package to assess its safety for humans. Additionally, we do not allow pre-compiled binaries for security. To request the inclusion of a package in the NextFerret repository, create an issue in the Debian user's repository, add the project's source code, and include the "package request" label. Then, just wait for the tests.

Regarding packages, we compile the first 30; if there is a total of 60, we wait for the first 30 to finish compiling and testing before starting the compilation of the next 30. And no, the packages are not frozen.
